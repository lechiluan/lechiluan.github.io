# Làm chủ Git trong Workflow dự án

Git không chỉ là công cụ để `commit` và `push`. Để trở thành một Senior Developer, bạn cần hiểu sâu hơn về các quy trình làm việc.

## 1. Git Rebase vs Merge

Nhiều người sợ Rebase, nhưng nó là cách tốt nhất để giữ cho lịch sử commit của bạn sạch sẽ.

```bash
git checkout feature-branch
git rebase main
```

## 2. Cherry-pick

Khi bạn chỉ muốn lấy một commit cụ thể từ nhánh khác mà không muốn merge cả nhánh đó.

## 3. Quản lý Conflict

Conflict không đáng sợ, nó là một phần của sự cộng tác. Sử dụng các công cụ như VS Code Merge Editor để xử lý nhanh hơn. Bạn có hiểu không.
