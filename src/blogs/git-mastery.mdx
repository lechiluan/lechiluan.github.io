<h2>Làm chủ Git trong Workflow dự án</h2>
<p>
  Git không chỉ là công cụ để <code>commit</code> và <code>push</code>. Để trở
  thành một Senior Developer, bạn cần hiểu sâu hơn về các quy trình làm việc.
</p>

<h3>1. Git Rebase vs Merge</h3>
<p>
  Nhiều người sợ Rebase, nhưng nó là cách tốt nhất để giữ cho lịch sử commit của
  bạn sạch sẽ.
</p>
<pre>
  <code class="language-bash">git checkout feature-branch git rebase main</code>
</pre>

<h3>2. Cherry-pick</h3>
<p>
  Khi bạn chỉ muốn lấy một commit cụ thể từ nhánh khác mà không muốn merge cả
  nhánh đó.
</p>

<h3>3. Quản lý Conflict</h3>
<p>
  Conflict không đáng sợ, nó là một phần của sự cộng tác. Sử dụng các công cụ
  như VS Code Merge Editor để xử lý nhanh hơn.
</p>
